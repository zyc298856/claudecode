# PPT 写作风格指南

本文档规定 PPT 幻灯片内容的写作风格和措辞规范，确保输出内容简洁、专业、有说服力。

## 核心原则

### 1. 一页一观点

每页只传达一个核心信息，标题即观点。

**反面示例**：
- 搜索优化与分享功能规划（两个主题在一页）

**正面示例**：
- 搜索优化：Q1 最高优先级（一个主题）

### 2. 标题即结论

标题不应是笼统的类别名，而是本页的核心结论。

**反面示例**：
- 市场分析
- 技术方案
- 团队介绍

**正面示例**：
- 市场规模 200 亿，年增长 25%
- 采用微服务架构，6 个月完成迁移
- 核心团队来自 BAT，平均 10 年经验

### 3. 要点精简

每个要点不超过 15 字（中文）/ 8 词（英文），用短语而非长句。

**反面示例**：
- 我们的智能客服产品基于最新的大语言模型技术，能够准确理解用户的意图并提供精准的回答，准确率达到了 95% 以上

**正面示例**：
- 大模型对话引擎，准确率 95%+

### 4. 数据驱动

优先使用量化表达，让成果可衡量。每个数字都应有上下文。

**量化角度**：
- 数量：完成 N 个功能、服务 N 个客户
- 比例：提升 N%、降低 N%、覆盖 N%
- 时间：提前 N 天交付、响应时间 Xms
- 规模：支持 N 并发、处理 N 条/秒

**注意**：
- 只有用户提供的数据才可使用
- 不要编造数据，如无数据则保留定性描述
- 数字必须有上下文（"40%" → "同比增长 40%"）

### 5. 动词开头

要点以动词开头，突出动作和成果。

**常用动词**：
- 展示类：发布、上线、推出、亮相
- 成果类：完成、实现、突破、超越
- 优化类：优化、提升、降低、缩短
- 规划类：规划、启动、推进、布局

## 各类型语气指南

### 商业提案（business-proposal）

- **语气**：自信、有说服力、专业
- **关键词**：解决方案、投资回报、竞争优势、行动号召
- **叙事结构**：痛点 → 方案 → 证明 → 行动
- **避免**：犹豫性语言（"可能"、"也许"、"大概"）

### 产品发布（product-launch）

- **语气**：热情、自信、富有感染力
- **关键词**：创新、突破、体验、里程碑
- **叙事结构**：痛点 → 亮相 → 功能 → 定价 → 号召
- **避免**：过于技术化的表述（面向普通受众时）

### 季度复盘（quarterly-review）

- **语气**：客观、数据驱动、务实
- **关键词**：达成、增长、挑战、下一步
- **叙事结构**：概览 → 细节 → 挑战 → 规划
- **避免**：夸大成绩、回避问题

### 培训材料（training-material）

- **语气**：清晰、亲切、鼓励互动
- **关键词**：学习目标、实操、练习、回顾
- **叙事结构**：目标 → 概念 → 示例 → 练习 → 回顾
- **避免**：过多术语、过快的节奏

### 技术演示（technical-presentation）

- **语气**：精确、逻辑清晰、有深度
- **关键词**：架构、方案、性能、权衡
- **叙事结构**：动机 → 方案 → 实现 → 结果 → 经验
- **避免**：过度简化、省略技术细节

### 项目启动（project-kickoff）

- **语气**：积极、明确、有凝聚力
- **关键词**：目标、里程碑、分工、协作
- **叙事结构**：背景 → 目标 → 计划 → 团队 → 行动
- **避免**：模糊的时间线、不清晰的责任划分

### 团队汇报（team-meeting）

- **语气**：简洁、务实、以成果为导向
- **关键词**：完成、进行中、计划、需要支持
- **叙事结构**：概览 → 重点 → 成果 → 挑战 → 规划
- **避免**：流水账、无重点的罗列

## 演讲者备注写作规则

演讲者备注是对幻灯片内容的**补充说明**，不是重复。

### 写什么
- 背景故事和上下文
- 具体案例和细节
- 数据的来源和计算方式
- 预判听众可能的疑问并准备回答

### 不写什么
- 不要重复幻灯片上已有的文字
- 不要写"这一页讲的是..."
- 不要超过 4 句话

### 示例

**幻灯片内容**：
```
## 搜索优化：Q1 最高优先级
- 用户反馈占比 35%
- 预计 2 周开发 + 1 周测试
- UI 改版同步推进
```

**好的演讲者备注**：
> 用户调研中，搜索相关的投诉远超其他问题。竞品 A 在上个月刚更新了搜索功能，我们需要尽快跟进。UI 团队已经完成了设计稿初版，可以和后端并行。

**差的演讲者备注**：
> 这一页介绍了搜索优化的优先级和计划。搜索优化是 Q1 最高优先级，占用户反馈的 35%，预计 2 周开发。

## 双语规则

- 中文输入 → 中文输出（幻灯片内容和演讲者备注均为中文）
- 英文输入 → 英文输出
- 术语处理：技术术语保留英文原文（API、K8s、CI/CD），不强制翻译
- 布局建议可使用中英混合（如"左右布局"）

## 常见错误

### 内容过载
- 每页超过 5 个要点 → 删减或拆分
- 要点超过 15 字 → 提炼核心，细节放演讲者备注
- 每页多主题 → 拆分为多页

### 缺乏数据支撑
- "效果显著" → "转化率提升 35%"
- "性能大幅优化" → "响应时间从 2s 降至 800ms"
- 如无数据，使用定性描述而非空洞形容词

### 被动语态
- "被优化了" → "优化了..."
- "被部署到了线上" → "已上线部署"
- "是一个解决方案" → "提供解决方案"
